🚀 HOW TO USE

Step 1: Install dependencies
  $ pip install -r requirements.txt

Step 2: Configure API keys
  $ cp .env.template .env
  $ nano .env  # Add your API keys

Step 3: Run the server
  $ python main.py

Step 4: Open dashboard
  → http://localhost:8000/dashboard

Step 5: Configure Vapi
  → Follow VAPI_SETUP_GUIDE.md

Step 6: Test
  $ python test_system.py

💡 ORDER FLOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Customer calls → Vapi answers
2. AI greets: "Hej, Gislegrillen! Vad får det lov att vara?"
3. Customer orders: "En Hawaii och en Kebabpizza"
4. AI confirms and asks for drinks
5. Customer confirms order
6. AI calls /place_order endpoint
7. Backend:
   • Calculates price
   • Saves to orders.json
   • Prints kitchen ticket
   • Sends Pushover notification
8. AI confirms: "Tack för din beställning! Den är klar om 15 minuter."
9. Kitchen staff sees order on dashboard
10. Staff marks order as ready
11. Customer picks up → Happy! 🍕

1. Add your API keys to .env file:
   • VAPI_API_KEY (from vapi.ai)
   • GROQ_API_KEY (from console.groq.com)
   • PUSHOVER_USER_KEY & PUSHOVER_API_TOKEN (from pushover.net)

2. Run the server:
   $ python main.py

3. Configure Vapi assistant:
   • Follow VAPI_SETUP_GUIDE.md step-by-step
   • Copy system_prompt.md to Vapi
   • Add place_order tool pointing to your server
   • Assign phone number

4. Test it:
   • Make a test call
   • Watch console for kitchen ticket
   • Check dashboard at http://localhost:8000/dashboard
   • Verify Pushover notification

5. Deploy to production (optional):
   • Use a cloud provider (AWS, DigitalOcean, Heroku)
   • Set up HTTPS
   • Configure proper domain
   • Update Vapi with production URL
